## 一、專案簡介

本次期末專案製作的是一款以 **C 語言** 撰寫的文字介面貪吃蛇遊戲。遊戲在終端機中執行，玩家可以透過鍵盤的 `W`、`A`、`S`、`D` 控制蛇的移動方向，目標是在避免撞牆與撞到自己身體的情況下，盡可能吃到更多食物並提高分數。

本專案主要功能如下：

1. **三種遊戲速度模式**  
   遊戲開始時，玩家可以選擇初級、中級或困難模式。不同模式會影響遊戲更新速度，困難模式的反應時間最短。

2. **蛇身移動與成長機制**  
   蛇會根據目前方向持續前進。當蛇吃到食物後，分數會增加，蛇身長度也會變長。

3. **碰撞判定**  
   遊戲會判斷蛇是否撞到牆壁或撞到自己的身體。一旦發生碰撞，遊戲就會結束。

4. **普通食物與黃金食物**  
   普通食物可以增加 10 分，黃金食物可以增加 50 分。黃金食物具有倒數時間，若玩家未在時間內吃到，食物會重新生成。

5. **終端機彩色畫面**  
   專案使用 ANSI escape code 呈現不同顏色，例如牆壁、蛇頭、蛇身、食物與遊戲資訊欄，使文字遊戲畫面更有視覺效果。

6. **遊戲結束動畫**  
   當遊戲結束時，程式會顯示簡單的 Game Over 動畫與最終分數。

本專案使用 `Makefile` 管理編譯流程，並將程式拆成多個 `.c` 與 `.h` 檔案，包括 `main.c`、`game_core.c`、`logic.c`、`renderer.c`、`io_utils.c` 等，使程式結構比單一檔案更清楚。

## 二、專案架構與檔案分工

本專案採取模組化設計，將不同功能分散到不同檔案中，降低單一檔案過於複雜的問題。

| 檔案 | 功能說明 |
|---|---|
| `main.c` | 負責遊戲主流程、開始選單、速度選擇、主遊戲迴圈與輸入方向控制 |
| `game_core.c / game_core.h` | 定義遊戲狀態，例如蛇的位置、蛇長度、方向、食物、分數與是否遊戲結束 |
| `logic.c / logic.h` | 負責遊戲邏輯，包括蛇移動、吃食物、加分、撞牆、撞身體、黃金食物倒數 |
| `renderer.c / renderer.h` | 負責畫面輸出，包括遊戲地圖、蛇、食物、分數欄與結束動畫 |
| `io_utils.c / io_utils.h` | 處理鍵盤輸入與延遲功能，例如非阻塞按鍵偵測與毫秒級暫停 |
| `Makefile` | 管理專案編譯流程，使用 `make` 即可編譯專案 |

整體架構如下：

```text
main.c
  ├── 呼叫 Setup() 初始化遊戲
  ├── 呼叫 Draw() 繪製畫面
  ├── 呼叫 Input() 讀取玩家輸入
  ├── 呼叫 Logic() 更新遊戲邏輯
  └── 呼叫 SleepMs() 控制遊戲速度

game_core.c
  └── 管理遊戲狀態資料

logic.c
  └── 處理蛇移動、碰撞、吃食物與分數

renderer.c
  └── 負責終端機畫面顯示

io_utils.c
  └── 處理鍵盤輸入與時間延遲
```

這樣的分工讓專案比較容易閱讀與維護，也比較符合程式設計中「分層」與「模組化」的概念。

## 三、開發過程

### 1. 專案規劃階段

一開始，我先決定製作一個難度適中、可以完整展示程式邏輯的文字小遊戲。貪吃蛇雖然是經典遊戲，但它包含許多程式設計的重要觀念，例如：

- 陣列資料結構
- 迴圈控制
- 條件判斷
- 即時輸入
- 遊戲主迴圈
- 座標系統
- 碰撞判定
- 隨機數生成
- 模組化程式設計

因此，我選擇以 C 語言完成貪吃蛇遊戲，並且不只寫出基本版本，而是加入速度選擇、黃金食物、分數顯示、彩色終端機畫面與結束動畫，使專案成果更完整。

---

### 2. 程式架構建立

在開發初期，如果把所有程式都寫在 `main.c` 裡，雖然可以快速完成，但後期會變得難以維護。因此我將專案拆成數個模組。

`main.c` 主要負責遊戲流程，核心迴圈如下：

```c
while (!gameOver) {
    Draw();
    Input();
    Logic();
    SleepMs(gameDelay);
}
```

這段程式是整個遊戲的核心。每一次迴圈都會依序執行：

1. `Draw()`：更新畫面  
2. `Input()`：讀取玩家輸入  
3. `Logic()`：更新遊戲邏輯  
4. `SleepMs(gameDelay)`：控制遊戲速度  

這樣的寫法讓主流程非常清楚，也讓其他功能可以分別放在不同模組中處理。

---

### 3. 遊戲資料設計

在 `game_core.h` 中，我定義了遊戲地圖大小：

```c
#define WIDTH 40
#define HEIGHT 20
```

也定義了方向、食物種類與座標結構：

```c
typedef enum { UP, DOWN, LEFT, RIGHT } Direction;
typedef enum { NORMAL, GOLDEN } FoodType;

typedef struct {
    int x;
    int y;
} Position;
```

蛇的身體使用陣列儲存：

```c
Position snake[500];
```

每一節蛇身都用一組 `(x, y)` 座標表示。蛇頭是 `snake[0]`，後面的陣列元素則代表蛇身。

這種寫法雖然簡單，但很適合用來實作貪吃蛇，因為蛇每次移動時，只需要讓後一節身體移動到前一節身體原本的位置即可。

---

### 4. 蛇的移動邏輯

蛇身移動的主要程式如下：

```c
for (int i = snakeLength - 1; i > 0; i--) {
    snake[i] = snake[i - 1];
}
```

這段程式的意思是，從蛇尾開始，每一節蛇身都移動到前一節原本的位置。

接著再根據目前方向移動蛇頭：

```c
switch (dir) {
    case UP:    snake[0].y--; break;
    case DOWN:  snake[0].y++; break;
    case LEFT:  snake[0].x--; break;
    case RIGHT: snake[0].x++; break;
}
```

這樣就能形成蛇連續前進的效果。

---

### 5. 碰撞判定

遊戲中有兩種主要碰撞判定。

第一種是撞牆：

```c
if (snake[0].x == 0 || snake[0].x == WIDTH - 1 ||
    snake[0].y == 0 || snake[0].y == HEIGHT - 1) {
    gameOver = true;
}
```

第二種是撞到自己：

```c
for (int i = 1; i < snakeLength; i++) {
    if (snake[0].x == snake[i].x && snake[0].y == snake[i].y) {
        gameOver = true;
    }
}
```

只要蛇頭的位置與牆壁或任一節蛇身重疊，就會觸發遊戲結束。

---

### 6. 食物與分數機制

當蛇頭位置與食物位置相同時，代表吃到食物：

```c
if (snake[0].x == food.x && snake[0].y == food.y) {
    if (foodType == GOLDEN) {
        score += 50;
    } else {
        score += 10;
    }
}
```

普通食物加 10 分，黃金食物加 50 分。

同時，程式會將蛇的長度增加一節：

```c
if (snakeLength < 500) {
    snake[snakeLength] = prevTail;
    snakeLength++;
}
```

這段程式會把原本的蛇尾位置補回去，使蛇身變長。

黃金食物則是使用隨機機率產生：

```c
if (rand() % 5 == 0) {
    foodType = GOLDEN;
    bonusTimer = 35;
} else {
    foodType = NORMAL;
}
```

也就是每次吃到食物後，大約有 20% 機率產生黃金食物。

## 六、問題與解決方法

### 問題一：多檔案專案的結構不熟悉

一開始我比較熟悉單一 `main.c` 的寫法，但這次專案需要拆成多個檔案，因此遇到 `.h` 檔、`.c` 檔、`extern` 宣告與編譯順序的問題。

解決方法是先將資料結構與函式宣告放在 `.h` 檔，實際函式內容放在 `.c` 檔。例如 `game_core.h` 負責宣告遊戲狀態，`game_core.c` 負責真正建立變數與初始化資料。

最後再用 `Makefile` 統一編譯，使整個專案可以透過以下指令完成編譯：

```bash
make
```

這讓我理解 C 語言多檔案專案不是單純把程式拆開，而是要正確處理宣告、定義與連結。

---

### 問題二：蛇身移動邏輯容易寫錯

貪吃蛇的移動不是只移動蛇頭，還必須讓每一節蛇身跟著前一節移動。若順序寫錯，蛇身可能會重疊或移動異常。

最後採用從蛇尾往蛇頭方向更新的方式：

```c
for (int i = snakeLength - 1; i > 0; i--) {
    snake[i] = snake[i - 1];
}
```

這樣可以避免前面的座標資料被過早覆蓋，讓蛇身移動保持正確。

---

### 問題三：即時鍵盤輸入處理不容易

一般 C 語言的 `scanf()` 需要按 Enter 才會輸入，不適合即時遊戲。因此本專案使用 `termios` 和 `fcntl` 製作非阻塞輸入。

相關功能放在 `io_utils.c` 中：

```c
int CustomKbhit(void);
int CustomGetch(void);
```

這讓遊戲可以在不中斷主迴圈的情況下偵測玩家是否按下按鍵。

不過這種寫法比較適合 Linux、macOS 或 WSL 環境，在 Windows 原生環境下相容性較差。這也是本專案目前的限制之一。

---

### 問題四：食物可能生成在蛇身上

在專案中，吃到食物後會重新生成新食物。若沒有檢查新食物位置是否和蛇身重疊，就可能造成食物出現在蛇身裡，影響遊戲公平性。

目前在吃到食物後，已經使用迴圈檢查食物是否在蛇身上：

```c
bool onSnake;
do {
    onSnake = false;
    food.x = rand() % (WIDTH - 2) + 1;
    food.y = rand() % (HEIGHT - 2) + 1;

    for (int i = 0; i < snakeLength; i++) {
        if (snake[i].x == food.x && snake[i].y == food.y) {
            onSnake = true;
            break;
        }
    }
} while (onSnake);
```

這樣可以讓新食物避開蛇身。

不過目前仍有可改進之處：初始化食物與黃金食物過期後重新生成食物時，也應該使用相同的檢查邏輯。比較好的做法是把這段程式整理成 `SpawnFood()` 函式，讓所有產生食物的地方都呼叫同一個函式，避免重複程式與遺漏檢查。

---

### 問題五：畫面刷新方式較粗糙

目前 `renderer.c` 中使用：

```c
system("clear");
```

來清除畫面。這樣可以達到效果，但不是最理想的做法，因為它依賴外部指令，而且每次刷新都呼叫系統指令，效率較差。

未來可以改成 ANSI escape code：

```c
printf("\033[H\033[J");
```

這樣可以直接控制終端機畫面，效率較好，也比較符合文字介面遊戲的寫法。

## 八、學習心得

透過這次專案，我學到最多的是：完成一個程式不只是讓它能執行，還要考慮程式架構、可讀性、可維護性與後續擴充性。

在過去寫小程式時，常常會把所有內容都寫在 `main.c` 中。這樣在程式很短的時候沒有問題，但當功能越來越多，例如加入畫面、輸入、分數、食物、碰撞判定與動畫後，單一檔案就會變得很難閱讀。因此這次專案讓我更理解模組化的重要性。

這次也讓我更熟悉 C 語言中的標頭檔與多檔案編譯。`.h` 檔不是拿來放主要程式內容，而是用來宣告函式、結構、列舉與外部變數；`.c` 檔則負責實作功能。這個觀念對 C 語言專案很重要。

AI 對本次專案有明顯幫助，尤其是在架構規劃與除錯方面。AI 可以快速提出一個基本方向，也能指出程式中可能存在的邏輯漏洞，例如食物生成是否會與蛇身重疊、遊戲輸入是否穩定、`system("clear")` 是否適合長期使用等。這些提醒讓我更容易從「只是寫出能跑的程式」進一步思考「如何寫得更好」。

不過，AI 也有明顯限制。AI 給出的程式不一定完全適合自己的環境，也不一定一開始就完全正確。尤其 C 語言牽涉到作業系統差異，例如 `termios` 適合 Linux 或 macOS，但不一定能在 Windows 原生環境中使用。因此不能完全依賴 AI 產生程式後就直接交出，而是必須自己理解、測試與修改。

我認為這次專案最大的收穫是：AI 可以加快開發速度，但真正重要的是自己要能判斷 AI 的答案是否合理。AI 比較像是一個輔助工具，可以幫忙產生範例、整理架構、提示錯誤方向，但最後的理解、測試與取捨仍然需要自己完成。

本次專案中，我有使用生成式 AI 作為輔助工具，但並非完全由 AI 自動完成。

### 自己完成或主導的部分

1. 決定專案主題為 C 語言貪吃蛇遊戲

### AI 協助的部分

## 十、未來改進方向

雖然本專案已經完成基本功能，但仍有幾個可以進一步改進的方向。

### 1. 整理食物生成邏輯

可以將食物生成邏輯整理成獨立的 `SpawnFood()` 函式。這樣初始化食物、吃到食物後生成新食物、黃金食物過期後生成新食物，都可以使用同一套邏輯，避免食物出現在蛇身上。

### 2. 減少全域變數

目前蛇的位置、分數、方向、食物種類、遊戲狀態都使用全域變數。雖然小型專案可以接受，但如果未來要擴充，建議改成 `GameState` 結構，讓遊戲狀態集中管理。

範例：

```c
typedef struct {
    Position snake[500];
    int snakeLength;
    Direction dir;
    Position food;
    FoodType foodType;
    int bonusTimer;
    int score;
    bool gameOver;
    int gameDelay;
} GameState;
```

### 3. 改善畫面刷新方式

目前使用 `system("clear")` 清除畫面，未來可以改用 ANSI escape code，提高效率並減少對系統指令的依賴。

### 4. 提高跨平台相容性

目前鍵盤輸入方式主要適合 Linux、macOS 或 WSL。如果要讓 Windows 使用者也能直接執行，需要另外加入 Windows 版本的輸入處理。

### 5. 增加更多遊戲功能

未來可以加入：

- 排行榜
- 暫停功能
- 關卡制度
- 障礙物
- 不同食物效果
- 存檔紀錄
- 更完整的開始選單

這些功能可以讓遊戲從基本版本進一步變成更完整的小型遊戲專案。
